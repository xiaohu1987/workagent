# CodeXH

CodeXH 是一个本地桌面 Agent 工作台。它把聊天、代码工具、项目文件、终端、内置浏览器、知识库、Skill 与 MCP 服务放在同一个任务界面中，用于完成代码实现、资料检索、网页操作和本地工作流。

应用以 Electron 桌面程序运行。聊天内容和运行记录保留在本机；模型服务、MCP 服务和项目目录由用户自行配置和授权。

## 适用场景

- 在已选项目目录中阅读、修改、创建和验证代码。
- 通过受控工具执行终端命令、搜索代码、运行本地服务。
- 在右侧内置浏览器中打开网页、读取页面、截图或进行受控自动化。
- 导入本地文档，按片段检索并在回答中标注知识库来源。
- 把重复性流程封装成 Skill，或接入 MCP 服务提供额外能力。
- 将多条需求在任务执行时排队，按顺序自动继续。

## 快速开始

### 运行环境

开发运行需要：

- Windows 10/11。
- Node.js 与 pnpm。
- 已安装项目依赖：`pnpm install`。

### 启动应用

推荐使用根目录的启动脚本：

```bat
start-codexh.bat
```

脚本会在启动前重新构建 `dist`，然后关闭本项目旧的 Electron 实例并启动最新版本。开发模式可使用：

```powershell
pnpm dev
```

仅构建桌面端：

```powershell
pnpm build
```

### 第一次配置

1. 打开设置，进入“模型/供应商”页。
2. 新增或编辑供应商，填写 API 地址、密钥和必要请求头。
3. 添加模型并打开该模型实际支持的能力，例如工具调用、多模态输入或生图。
4. 点击“测试”。测试会验证连接、原生工具调用、工具结果回传和最终回复。
5. 保存配置后，在聊天输入区选择供应商与模型。

模型状态含义：

| 状态 | 含义 |
| --- | --- |
| `Agent 已验证` | 已通过完整工具调用往返测试，适合 Agent/GPA。 |
| `未验证 Agent` | 旧配置或尚未测试的模型；若开启了工具调用，可正常尝试工具。建议先测试。 |
| `仅聊天` | 模型关闭工具调用，或运行中多次协议失败后被自动降级；不应用于 GPA/工具任务。 |

## 界面与基本操作

### 左侧：任务历史

- 新建普通聊天：适合问答、检索和临时任务。
- 新建项目聊天：选择一个项目文件夹；后续文件工具默认在该文件夹内执行。
- 可通过历史搜索快速定位任务，支持置顶和删除。
- 项目级聊天会显示项目标识；普通聊天不绑定项目目录。

### 中间：聊天与输入区

- 输入消息后点击绿色“发送”。
- 任务运行中：输入框为空时显示红色“停止”；输入文字或附件后仍显示绿色“发送”，新内容会进入队列，不会打断当前任务。
- 队列按 FIFO 顺序执行。每项可删除；应用重启后仍会恢复未执行项。
- 运行状态可展开，查看公开的执行步骤、工具名称、参数摘要、结果和耗时。模型内部推理不会显示。
- 用户消息支持复制和重新编辑；代码块和长文本可一键复制。

### 右侧：工作区

右侧可在以下标签页切换：

- **文件**：浏览当前项目文件并预览文本内容。
- **终端**：Agent 的 `shell.exec` 命令与用户输入均在该会话终端中执行。
- **浏览器**：查看网页、浏览器标签和 Agent 当前打开的页面。
- **预览**：查看项目或可预览内容。

右侧面板可收起，浏览器和终端支持多个标签/会话。

## 模型与 Agent 工具

### 支持的供应商类型

当前适配器包含：

- OpenAI Compatible / Gateway / OpenRouter / Ollama / vLLM。
- Anthropic。
- Gemini。
- Mock（用于本地开发验证）。

当模型和供应商支持原生 Function Calling 时，CodeXH 使用供应商原生工具协议，而不是要求模型在普通回复中拼接工具 JSON。

### 工具调用流程

一次工具任务通常经过：

1. 模型收到当前聊天、项目上下文和可用工具。
2. 模型通过原生调用通道请求工具。
3. 程序校验工具是否存在、参数是否满足 Schema、是否需要审批。
4. 工具在项目目录、终端、浏览器或 MCP 中执行。
5. 结果以原生工具结果格式回传模型。
6. 模型继续调用工具或输出最终总结。

工具参数会校验必填字段、类型、数组项、枚举和联合格式。参数不合法时不会执行工具，也不会触发审批；模型会收到可修正的错误信息。

### 工具失败与模型降级

- 同一工具任务连续失败时，运行时会要求模型改变策略，而不是重复相同调用。
- 模型响应超时会自动重试，界面显示重试状态。
- 工具协议连续失败两次时，当前任务会停止并提示切换模型。
- 此类模型会自动标记为“仅聊天”，记录原因到配置和日志；界面会实时更新状态。

## GPA 计划执行

GPA 是 Goal / Plan / Act 的计划执行模式，适合范围较大、需要先确认实施方案的任务。

### 使用方式

1. 在输入区打开 GPA。
2. 发送需求，模型先生成目标理解和计划。
3. 确认计划后进入 ACT 执行阶段。
4. 执行过程中如遇到会影响范围、优先级、选型或验收的重大不确定项，模型可显示澄清卡。
5. 选择一个选项、填写补充说明或跳过；模型根据答案继续执行。

GPA 的澄清使用 `request_user_input` 工具。等待用户回答时，其他工具不会与澄清混在同一批执行，避免任务在等待中继续修改文件。

GPA 是一次性模式：任务完成、失败或手动停止后，输入区的 GPA 状态会清除。

## 项目、文件与任务输出

### 项目聊天

项目聊天必须选择文件夹。Agent 的文件读取、补丁和终端命令默认以该目录为工作目录。

### 无项目聊天

未选择项目时，临时任务输出不会写入程序安装目录，而是写入：

```text
%USERPROFILE%\.codexh\outputs\<threadId>\
```

### 输出文件

- 项目聊天的任务输出位于 `<项目目录>\.codexh\outputs\<threadId>\`。
- 生成图片、浏览器快照和其他 artifact 会纳入任务输出。
- 任务总结后会展示本次编辑或生成的文件列表。
- 本地路径可一键打开文件或所在文件夹；HTTP 链接可在浏览器中打开。

## 工具、终端与浏览器

### 内置工具

常用内置能力包括：

- 文件和目录读取。
- 代码搜索，优先使用 `rg`，不可用时回退 `grep`。
- `apply_patch` 文件创建/修改。
- 终端命令执行。
- Git 状态、差异、提交和 worktree 操作。
- 本地知识库检索与读取。
- 网络搜索和网页读取。
- 浏览器标签、页面检查、点击、填写、滚动、等待和截图。

写文件优先使用 `apply_patch`。不存在 `create_file` 工具；若模型尝试调用它，运行时会返回明确错误。

### 终端

终端是任务级会话：

- Agent 的命令和用户手动输入使用同一终端会话。
- 长时间运行的本地服务应使用后台启动方式，使工具调用本身能完成。
- 发现本地服务 URL 后，应用会在右侧浏览器中打开对应的 `http://127.0.0.1:<port>` 地址。

### 内置浏览器

浏览器工具遵循“先检查，再操作”的流程：

1. 使用页面检查获取可访问元素与元素 ID。
2. 对指定元素点击、填写、选择、滚动或按键。
3. 页面变化后重新检查。

普通读取、导航、滚动和填写可直接执行；提交表单、发送消息、下载、上传、支付、删除和外部协议跳转需要审批。

## 本地知识库

知识库采用本地文档分块与 SQLite FTS 检索，不依赖向量数据库或联网模型。

### 导入

设置中的“知识库”页分别提供“选择文件”和“选择文件夹”。文件夹会递归扫描支持格式，跳过隐藏、不支持或重复文件。

支持的文档会被解析为文档和片段，并保留标题、来源路径、页码/工作表/幻灯片等定位信息。

### 范围

| 范围 | 可见对象 | 删除项目/聊天后的行为 |
| --- | --- | --- |
| 全局 | 所有聊天 | 仍保留，直到在知识库管理中删除。 |
| 项目 | 同项目的聊天 | 知识库数据仍在本地，可在管理页删除。 |
| 仅当前对话 | 导入时所在聊天 | 对话删除后不再被该聊天使用；可在管理页清理。 |

### 使用

输入区打开“知识库”后，该聊天才会向模型暴露 `knowledge.search` 与 `knowledge.read`。关闭后不会检索知识库。

模型应先搜索相关片段，再按需读取；最终回答会标注引用知识库来源。

知识库管理页支持查看文档、刷新变更、重新索引和删除。刷新按源文件哈希进行增量处理。

## Skill 与 MCP

### Skill

Skill 是带有 `SKILL.md` 的工作说明或领域流程。可在设置的 Skill 管理页查看来源和说明，也可在输入区的“添加”菜单选择。

选择 Skill 后，它会作为当前消息的上下文；模型需要时可通过 `skills.load` 读取完整指令。

### MCP

MCP 管理页支持：

- 新增、编辑、删除、启停服务。
- `stdio` 配置：命令、参数、环境变量、工作目录。
- SSE / Streamable HTTP 配置：URL。
- 连接测试及工具、资源、资源模板查看。

MCP 工具按动态发现方式工作：模型先调用 `mcp.list_tools` 获取当前线程可访问的目录，再用 `mcp.call` 调用具体服务工具。高风险调用继续遵循审批策略。

插件提供的 MCP 服务在管理页只读显示，是否可用于项目由项目插件启用状态决定。

## 附件、多模态与生图

### 输入附件

支持：

- 粘贴截图。
- 拖入一个或多个图片或文件。
- 文件选择。

附件会复制到本地稳定目录：

```text
%USERPROFILE%\.codexh\attachments\<threadId>\
```

图片在输入区显示缩略图，文件显示标签，均可单独移除。图片会按不同供应商协议发送给支持多模态输入的模型；普通文件作为稳定路径和附件上下文提供，不会自动全文注入。

不支持多模态输入的模型会在发送前提示并阻止不兼容的图片消息。

### 生图

为模型打开“支持生图”后，发送消息会走 OpenAI Compatible 的 `/images/generations` 接口。结果保存为任务输出 artifact，并在聊天中以图片卡片展示。支持返回 `b64_json` 或 URL 的服务。

## 权限与安全

设置中的审批模式决定高风险操作如何处理。高风险范围包括但不限于：

- 写入或删除文件。
- 执行终端命令。
- Git 提交和 worktree 操作。
- MCP 高风险调用。
- 浏览器表单提交、发送、上传、下载和外部协议跳转。

审批卡可选择单次允许、当前会话允许或记住规则（具体选项取决于操作类型）。建议只对已确认的项目目录和命令授予长期权限。

应用只允许文件工具在项目目录或该聊天输出目录中访问；附件和输出预览也通过受控 IPC 读取，渲染进程不会直接读取任意本地路径。

## 本地数据目录

默认应用目录为：

```text
%USERPROFILE%\.codexh\
```

主要内容：

| 路径 | 内容 |
| --- | --- |
| `config.toml` | 供应商、模型、MCP 和桌面设置。请妥善保护其中的密钥。 |
| `codexh.sqlite` | 聊天、任务、工具记录、知识库索引等 SQLite 数据。 |
| `attachments\` | 聊天导入的图片与文件副本。 |
| `outputs\` | 无项目聊天的任务输出和 artifact。 |
| `logs\runtime.jsonl` | 运行日志。 |

项目聊天的产物默认在项目目录的 `.codexh\outputs\` 下。

## 日志与故障排查

日志文件：

```text
%USERPROFILE%\.codexh\logs\runtime.jsonl
```

建议先搜索以下事件：

| 日志事件 | 说明 |
| --- | --- |
| `provider.turn_timeout` | 模型请求超时及重试。 |
| `agent.model_protocol_failure` | 模型未返回有效工具协议。 |
| `model.agent_capability_downgraded` | 模型被自动降级为仅聊天。 |
| `tool.execution_error` | 工具执行异常。 |
| `turn.repeated_task_failure` | 同一工具任务连续失败达到阈值。 |
| `agent.context_compacted` | 上下文接近上限，已执行压缩。 |

### 常见问题

#### 模型不调用任何工具

1. 在设置中确认模型已打开“支持工具调用”。
2. 模型状态若为“仅聊天”，查看状态提示或日志，重新测试模型或切换模型。
3. 旧配置显示“未验证 Agent”时，只要模型支持工具调用，仍可尝试工具；建议执行一次模型测试并保存。
4. 检查当前任务是否处于 GPA 的 Goal/Plan 阶段。该阶段只进行分析，默认不执行文件或终端工具。

#### `Tool execution failed: Unknown tool: create_file`

程序没有 `create_file` 工具。应使用 `apply_patch` 创建文件，或让模型调用已列出的文件工具。

#### 工具调用后反复循环

查看日志中的工具名称、参数和失败结果。相同失败调用会被限制，运行时会要求模型更换策略；如果是模型协议问题，重新测试模型并切换到通过工具往返测试的模型。

#### 终端命令一直显示执行中

长驻服务必须后台启动，或使用会释放终端的命令。命令输出中出现本地 URL 后，可在右侧浏览器打开该 URL。

#### 浏览器打开了网页但模型没有获得内容

让模型先调用页面检查或读取页面文本。受反爬、登录、验证码或跨域 iframe 限制时，需要用户在右侧浏览器接管操作。

#### MCP 服务不可用

在 MCP 管理页执行连接测试，检查 transport、命令/URL、环境变量和网络。MCP 工具必须先通过 `mcp.list_tools` 被发现，之后才能调用。

## 开发与验证

常用命令：

```powershell
pnpm test
pnpm typecheck
pnpm build
```

当前测试覆盖原生工具调用、工具结果关联、多工具上下文压缩、GPA 澄清、参数校验、MCP 参数、浏览器参数、模型能力与恢复策略等关键路径。

## 使用建议

- 代码任务优先创建项目聊天，并明确给出验收标准。
- 长任务先启用 GPA，让计划经过确认后再执行。
- 使用前测试模型；适合普通聊天的模型不一定适合工具调用。
- 知识库仅在需要本地资料时开启，减少无关上下文。
- 对写文件、命令、上传和网页提交保留审批，尤其不要对不明命令选择长期允许。
- 任务失败时优先查看运行详情和 `runtime.jsonl`，不要盲目重复发送相同请求。
