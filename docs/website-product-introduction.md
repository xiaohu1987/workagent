# CodeXH 产品介绍资料

## 1. 产品定位

**产品名称：** CodeXH  
**一句话介绍：** 把聊天、代码执行、项目文件、终端、浏览器、知识库、Skill 与 MCP 服务放进同一个本地桌面工作台，让 AI 能在清晰授权下完成真实任务。  
**产品类型：** Windows 桌面 Agent 工作台。  
**核心理念：** 本地工作区、过程可见、关键操作可确认、结果可追溯。

CodeXH 面向需要让 AI 参与实际工作的个人开发者、技术团队和专业用户。它不是只输出建议的聊天窗口，而是围绕一个任务把需求、项目上下文、工具调用、执行记录、文件产物和验证过程组织在一起。

## 3. 面向用户的价值

### 从“建议”到“交付”

AI 可以理解任务和项目上下文，使用文件、终端、浏览器和外部服务完成工作，并在结束时呈现本次修改或生成的结果。

### 复杂任务先计划，再执行

GPA（Goal / Plan / Act）将复杂工作拆成目标澄清、计划确认和执行验证三个阶段。遇到关键选择时，AI 暂停等待用户决定，不在等待期间继续修改。

### 本地工作区保持在掌控中

项目文件、聊天记录、任务输出、附件和知识库索引保存在本机。涉及写文件、执行命令、提交表单或高风险 MCP 调用时，可以按规则要求确认。

### 一个界面连接完整工作流

无需在聊天窗口、编辑器、终端、浏览器、文档与外部工具之间反复切换。CodeXH 把这些能力组织在同一任务视图中。

## 4. 核心能力卡片

### 项目 Agent

选择一个项目文件夹后，AI 以该目录为工作范围读取文件、搜索代码、应用补丁、执行命令，并输出修改摘要。

**适合：** 功能开发、修复问题、重构、测试、代码审查、脚本编写。

### GPA 计划执行

为跨文件、跨步骤或需要设计决策的任务提供 Goal、Plan、Act 流程。计划中的澄清问答、用户选择与任务进度都会保留在聊天记录中。

**适合：** 新功能、架构调整、复杂排障、长任务。

### 内置终端与浏览器

在任务内运行本地命令、启动开发服务、查看 URL；也可在右侧浏览器中阅读网页、检查页面、截图与执行受控操作。

**适合：** 本地调试、前端验收、资料检索、网页工作流。

### 本地知识库

导入文档或文件夹后，CodeXH 在本机建立可检索资料库。对话可按需开启知识库检索，并在回答中标注资料来源。

**支持格式：** Markdown、文本、JSON、HTML、CSV、Excel、Word、PDF、PowerPoint。

### Skills

将领域规则、团队流程或重复工作写成 Skill。选择 Skill 后，它会成为当前任务的专业上下文，帮助 AI 按既定方式完成工作。

**适合：** 代码规范、部署流程、文档模板、专项工作方法。

### MCP 服务

连接本地或远程 MCP 服务，把额外的工具、资源和资源模板带入任务。支持 stdio、SSE 与 Streamable HTTP，并提供连接测试和调用策略。

**适合：** 内部平台、数据库、设计工具、知识服务、专用自动化。

### 附件与多模态输入

支持粘贴截图、拖入图片或文件、选择本地文件。具备多模态能力的模型可以理解图片；生成图片结果会作为任务产物保存并在聊天中展示。

### 执行记录与产物

聊天中展示公开的执行步骤、工具名称、参数摘要、结果和耗时。任务完成后可查看本次修改和生成的文件，并直接打开文件或所在文件夹。

## 6. 推荐展示场景

### 场景一：修复一个项目问题

用户选择项目文件夹，描述现象并提供复现步骤。CodeXH 读取相关文件、搜索代码、运行测试，在需要写入时请求确认，最后给出修改文件与验证结果。

### 场景二：实现一个完整功能

用户开启 GPA，先让 AI 拆解目标与实施计划。对于技术选型、范围或验收标准，用户直接在聊天中选择答案；确认计划后，AI 按任务顺序实现并验证。

### 场景三：基于本地资料完成工作

用户导入产品文档、规范或项目资料，开启知识库后让 AI 基于资料生成方案、文档或代码。回答中可看到引用来源，便于复核。

### 场景四：连接团队专用工具

用户配置 MCP 服务，将内部平台或专用工作流接入当前任务。AI 会先发现可用工具，再按权限策略调用。

## 7. 功能事实与表述边界

以下表述可用于官网，并应保持准确：

- CodeXH 是桌面程序，当前发布目标为 Windows。
- 聊天内容、运行记录、附件、任务输出和知识库索引默认保存在本机目录。
- 调用模型时，任务内容会发送给用户配置的模型服务；调用 MCP 时，必要参数会发送给对应服务。官网不应表述为“所有数据永不离开本机”。
- 写文件、执行终端命令、Git 操作、高风险 MCP 调用，以及浏览器提交、上传、下载等操作可触发确认。
- Agent 能力依赖模型是否支持工具调用；模型测试会显示其是否适合 Agent/GPA。
- 浏览器自动化受登录、验证码、反爬和跨域 iframe 等网页限制影响，必要时需要用户接管操作。
- CodeXH 不承诺替代代码审查、生产变更审批或安全审计。
