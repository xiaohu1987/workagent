# CodeXH 使用文档

## 1. CodeXH 是什么

CodeXH 是本地优先的桌面 Agent 工作台。它将聊天、项目文件、终端、浏览器、知识库、Skills 和 MCP 服务整合到一个任务界面中，用于代码实现、资料检索、网页操作和本地工作流。

程序在本机运行。聊天记录和运行记录会保存到本机；模型服务、MCP 服务和项目目录需要由用户自行配置和授权。

## 2. 开始前准备

### 运行环境

- Windows 10 或 Windows 11。
- 可用的模型服务及 API 密钥，或本地模型服务。
- 如需执行代码任务，准备一个本地项目文件夹。

### 首次配置模型

1. 打开 **设置**，进入 **模型 / 供应商**。
2. 新增或编辑供应商，填写 API 地址、密钥和必要请求头。
3. 添加模型，并仅开启该模型实际支持的能力，例如工具调用、多模态输入或图片生成。
4. 点击 **测试**，确认连接、工具调用和结果回传正常。
5. 保存后，在聊天输入区选择供应商和模型。

模型状态说明：

| 状态 | 含义 | 建议 |
| --- | --- | --- |
| Agent 已验证 | 已通过工具调用往返测试 | 可用于 Agent 和 GPA 任务 |
| 未验证 Agent | 尚未完整测试，但已配置工具能力 | 可尝试使用，建议先测试 |
| 仅聊天 | 当前模型不适合工具调用，或已被自动降级 | 用于问答，不建议用于 GPA 或工具任务 |

## 3. 创建任务

### 普通聊天

适合问答、资料整理、临时分析和不需要绑定项目目录的任务。点击新建聊天后直接输入需求即可。

### 项目聊天

适合代码修改、测试和本地项目工作。

1. 新建项目聊天。
2. 选择项目文件夹。
3. 在输入框描述目标、限制和验收标准。
4. Agent 会以该项目目录作为文件和终端工具的默认工作范围。

建议在需求中说明：需要修改什么、不能修改什么、如何判断完成，以及是否需要运行测试或构建。

## 4. 使用 GPA 完成长任务

GPA 代表 Goal、Plan、Act，适合跨文件、步骤多或需要先确认方案的任务。

### 操作步骤

1. 在输入区开启 **GPA**。
2. 发送需求。Agent 会先理解目标并制定计划。
3. 查看计划，确认后进入 **ACT** 执行阶段。
4. 如遇到影响范围、技术选型、优先级或验收标准的关键问题，Agent 会暂停并显示选项。
5. 选择答案，必要时填写补充说明，再确认提交。
6. 执行结束后，查看任务状态、修改文件与验证结果。

### 查看已选答案

Plan 问答完成后，聊天记录会显示一条 **已询问 N 个问题** 的摘要。点击它可以展开查看每一个问题、已选答案及补充内容。

### 使用建议

- 在 Plan 阶段明确验收标准，例如“运行测试通过”“页面在移动端不溢出”。
- 遇到不希望自动决定的技术选择，直接要求 Agent 在执行前询问。
- GPA 等待你的回答时，不会继续执行其他工具操作。

## 5. 查看和控制任务执行

### 运行记录

任务运行时，聊天中会展示公开的执行步骤、工具名称、参数摘要、结果和耗时。模型内部推理不会显示。

### 任务队列

当一个任务正在执行时，继续输入文字或添加附件会将新需求加入队列，不会打断当前任务。队列按提交顺序执行，可以删除尚未执行的项目。

### 停止任务

任务运行时，清空输入框后可使用停止按钮中断当前任务。已完成的工具结果和项目文件修改会保留。

## 6. 项目、文件与产物

### 右侧工作区

右侧区域可在以下标签间切换：

- **文件**：浏览和预览当前项目文件。
- **终端**：查看 Agent 命令和手动输入命令的输出。
- **浏览器**：查看 Agent 打开的网页和页面状态。
- **预览 / 变更**：查看可预览内容及本次任务相关的文件变更。

### 任务输出位置

- 项目聊天：`<项目目录>\.codexh\outputs\<threadId>\`
- 普通聊天：`%USERPROFILE%\.codexh\outputs\<threadId>\`

生成图片、浏览器快照和其他任务产物会保存在对应输出目录中。任务完成后可以从聊天记录直接打开文件或所在文件夹。

## 7. 使用终端和浏览器

### 终端

Agent 与用户在同一个任务级终端会话中执行命令。若启动长时间运行的开发服务，请使用后台启动方式，让当前工具调用能够结束。

当命令输出本地服务地址时，可在右侧浏览器打开对应的 `http://127.0.0.1:<port>` 页面。

### 浏览器

Agent 可以打开网页、读取页面、检查元素、点击、填写、滚动和截图。对于表单提交、发送、上传、下载、支付、删除和外部协议跳转等高风险操作，程序会要求确认。

遇到登录、验证码、反爬或跨域 iframe 限制时，请在右侧浏览器中手动完成需要你接管的步骤。

## 8. 使用本地知识库

### 导入资料

1. 打开 **设置 > 知识库**。
2. 选择导入单个文件或文件夹。
3. 选择知识范围：全局、项目或仅当前对话。
4. 等待索引完成。

支持 Markdown、文本、JSON、HTML、CSV、Excel、Word、PDF 和 PowerPoint 等常见文档格式。

### 在对话中使用

在输入区开启 **知识库** 后，Agent 才能检索当前可见范围内的资料。建议在提问中说明资料主题和期望输出，例如：“根据已导入的产品规范，整理登录流程的验收清单。”

回答中的资料来源可以展开查看，便于核对。

## 9. 使用 Skills 和 MCP 服务

### Skills

Skill 是一套可复用的工作说明或领域流程。

1. 在设置的 **Skills** 页面查看或管理可用 Skill。
2. 在输入区点击添加菜单，选择 **Skills**。
3. 选择本次任务需要的 Skill。
4. 发送任务，Agent 会将其作为当前任务的专业上下文。

适合将团队规范、文档模板、测试流程、发布步骤或专业方法沉淀下来。

### MCP 服务

MCP 用于接入额外工具和资源。

1. 在设置的 **MCP** 页面新增服务，或导入服务配置。
2. 选择传输方式：stdio、SSE 或 Streamable HTTP。
3. 填写命令、URL、参数、环境变量或工作目录。
4. 启用服务并执行连接测试。
5. 在输入区添加菜单选择 **MCP 服务**，指定本次任务优先使用的服务。

高风险 MCP 调用仍然遵循当前审批规则。

## 10. 附件、图片与生成内容

### 添加附件

可以通过以下方式添加内容：

- 粘贴截图。
- 拖入图片或文件。
- 在输入区添加菜单中选择文件或图片。

图片将发送给支持多模态输入的模型；普通文件会作为任务上下文和本地附件保存。若当前模型不支持多模态输入，程序会提示并阻止发送不兼容的图片消息。

### 生成图片

为模型开启图片生成能力后，可在需求中直接描述需要的图片。生成结果会显示在聊天中，并保存为任务产物，可查看原图和打开所在文件夹。

## 11. 权限与安全

CodeXH 会根据审批规则处理高风险操作，常见范围包括：

- 写入或删除文件。
- 执行终端命令。
- Git 提交和 worktree 操作。
- 高风险 MCP 调用。
- 浏览器的提交、发送、上传、下载和外部跳转。

确认卡片通常支持单次允许、当前会话允许或记住规则，具体选项会随操作类型变化。只应对你已核实的项目、命令和服务授予长期权限。

## 12. 常见问题

### 为什么 Agent 只聊天，不调用工具？

检查当前模型状态。模型需要开启并实际支持工具调用；建议在设置中重新测试模型。GPA 的 Goal / Plan 阶段默认用于分析和规划，不会直接执行文件或终端工具。

### 浏览器验证一直在等待怎么办？

确认右侧浏览器已打开并可用。遇到登录、验证码或网页限制时，在浏览器中手动处理后再继续。需要跳过验证时，按提示选择跳过或重试。

### MCP 服务不可用怎么办？

在 MCP 管理页执行连接测试，检查传输方式、命令或 URL、环境变量、网络及服务自身日志。服务启用后，Agent 还需要先发现可用工具才能调用。

### 任务失败后如何排查？

先查看聊天中的执行记录和报错摘要。更详细的运行日志位于：

`%USERPROFILE%\.codexh\logs\runtime.jsonl`

模型超时、工具协议异常、工具执行错误和上下文压缩等事件都会记录在这里。

## 13. 本地数据位置

默认应用数据目录：

`%USERPROFILE%\.codexh\`

| 路径 | 内容 |
| --- | --- |
| `config.toml` | 供应商、模型、MCP 和桌面设置；其中可能包含密钥 |
| `codexh.sqlite` | 聊天、任务、工具记录与知识库索引 |
| `attachments\` | 导入的图片和文件副本 |
| `outputs\` | 普通聊天的任务输出与产物 |
| `logs\runtime.jsonl` | 运行日志 |

请妥善保护 `config.toml` 中的密钥，以及项目目录与应用数据目录中的敏感内容。
